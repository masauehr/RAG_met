# CLAUDE.md — RAG_met プロジェクト指示

## プロジェクト概要

RAG（検索拡張生成）を使った気象・業務データ対応のAIチャットボット構築プロジェクト。
Claude APIを利用し、ローカルデータをベクトルDBに格納して検索・回答を行う。

## コーディング規約

- **言語**: Python 3.x
- **インデント**: スペース4つ
- **コメント**: 日本語で記述
- **ファイル名**: スネークケース（例: `ingest.py`, `query_engine.py`）

## ディレクトリ役割

| ディレクトリ | 役割 |
|------------|------|
| `src/` | メインコード（取り込み・検索・アプリ） |
| `data/` | 元データ（テキスト、PDF、CSV等） |
| `db/` | ChromaDBのベクトルデータ保存先 |

## 重要ルール

1. `.env` ファイルは絶対にコミットしない（APIキー保護）
2. `db/` ディレクトリのベクトルDBはコミット対象外（`.gitignore`に追加）
3. `data/` に個人情報を含むファイルを置く場合は事前確認必須
4. Claudeへのプロンプトはシステムプロンプトとユーザープロンプトを明確に分離する

## 開発の優先順位

1. **フェーズ1**: データ取り込み・ベクトル化（`ingest.py`）
2. **フェーズ2**: 検索・回答生成（`query.py`）
3. **フェーズ3**: CLIインターフェース（`app.py`）
4. **フェーズ4**: Webインターフェース（オプション）

## 使用する主要ライブラリ

```
anthropic          # Claude API クライアント
langchain          # RAGフレームワーク
langchain-community
chromadb           # ベクトルDB（ローカル）
sentence-transformers  # 埋め込みモデル（ローカル使用時）
python-dotenv      # 環境変数管理
```

## プロンプト設計指針

- システムプロンプトに「提供されたコンテキストのみを使って回答する」制約を明示する
- コンテキストが不十分な場合は「わかりません」と答えさせる（ハルシネーション防止）
- 回答には参照元ドキュメント名を含める
